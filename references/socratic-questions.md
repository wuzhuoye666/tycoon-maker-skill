# 苏格拉底提问库

> [!info] Obsidian 使用说明
> 本文档作为 Obsidian 笔记的参考资源，使用标准 Markdown 格式，支持 Obsidian 的双向链接和标签功能。

## 核心原则

苏格拉底式提问的核心是**引导思考，而非提供答案**。通过提问帮助用户：
1. 发现自己的假设
2. 挑验假设的有效性
3. 探索新的可能性
4. 形成自己的结论

---

## 提问类型分类

### 1. 澄清问题类

用于明确用户真正关心的问题。

| 提问 | 适用场景 |
|------|---------|
| "你真正想解决的核心问题是什么？" | 问题描述模糊 |
| "这个问题对你来说为什么重要？" | 需要确认优先级 |
| "如果这个问题解决了，你会得到什么？" | 需要明确目标 |
| "你现在卡在哪一步？" | 需要聚焦痛点 |
| "你能用一句话描述这个问题吗？" | 问题过于复杂 |

---

### 2. 挑战假设类

用于检验用户的思维前提。

| 提问 | 适用场景 |
|------|---------|
| "你做出这个判断的前提假设是什么？" | 结论跳跃太快 |
| "如果这个假设是错的，会怎样？" | 需要验证假设 |
| "有没有可能情况正好相反？" | 视角单一 |
| "你的假设有什么证据支撑？" | 假设缺乏依据 |
| "这个假设在什么情况下不成立？" | 需要边界分析 |

---

### 3. 探索证据类

用于检验用户的信息基础。

| 提问 | 适用场景 |
|------|---------|
| "你的判断基于什么数据或信息？" | 结论缺乏数据 |
| "有没有可能你遗漏了一些关键信息？" | 信息不完整 |
| "你怎么知道这个信息是可靠的？" | 信息来源存疑 |
| "有没有相反的证据？" | 视角单一 |
| "如果我要验证这个判断，我应该做什么？" | 需要验证方案 |

---

### 4. 探索观点类

用于打开思维边界。

| 提问 | 适用场景 |
|------|---------|
| "如果站在用户角度，他们会怎么看？" | 视角局限在自己 |
| "竞争对手会怎么想这个问题？" | 缺少竞品视角 |
| "如果资金有限，你会怎么做？" | 资源假设过度 |
| "如果只有一周时间，你会优先做什么？" | 时间假设过度 |
| "有没有更简单的方式？" | 方案过于复杂 |

---

### 5. 探索后果类

用于帮助用户预见结果。

| 提问 | 适用场景 |
|------|---------|
| "最坏的情况会是什么？你能承受吗？" | 风险意识不足 |
| "最好的情况会是什么？如何放大它？" | 乐观不足 |
| "如果成功了，下一步是什么？" | 缺少长期规划 |
| "如果失败了，你有什么退路？" | 缺少备选方案 |
| "这件事不做会有什么后果？" | 需要评估必要性 |

---

### 6. 元认知类

用于帮助用户反思自己的思考过程。

| 提问 | 适用场景 |
|------|---------|
| "你现在是怎么思考这个问题的？" | 思考过程不清晰 |
| "你之前的想法是什么？为什么改变了？" | 需要追踪认知变化 |
| "什么信息会让你改变判断？" | 需要确定决策标准 |
| "你觉得自己还缺少什么认知？" | 需要识别盲点 |
| "这个问题你思考了多久？" | 需要评估投入 |

---

## 提问节奏控制

### 三段式提问

```
第一轮：澄清问题（1-2个提问）
第二轮：挑战假设（2-3个提问）
第三轮：探索方向（1-2个提问）
```

### 什么时候停止提问

用户出现以下信号时，停止提问，进入调研或行动阶段：

- 用户说"我想我明白了..."
- 用户开始自己提出方案
- 用户明确说"帮我调研一下..."
- 用户说"接下来我应该怎么做..."
- 用户情绪明显不耐烦（需要调整方式）

## 特殊场景提问

### 创业方向决策场景

1. "你真正热爱的是什么？为什么？"
2. "如果这件事注定失败，你还会做吗？为什么？"
3. "十年后，你希望这件事变成什么样子？"

### 商业模式验证场景

1. "用户为什么愿意付费？真正的理由是什么？"
2. "如果免费，用户还会用吗？"
3. "你的收入来源真的是可持续的吗？"

### 资源有限场景

1. "如果只能用10%的资源，你能做什么？"
2. "什么是最小可行的验证方式？"
3. "有什么是你现在就能做的，不需要额外资源？"

## 复述确认模板

每次提问后，使用以下模板复述确认：

```
我理解你的意思是 [复述用户回答]。
我的理解正确吗？
如果不对，请纠正我。
```

**示例**：
- 用户："我觉得这个市场太小了，可能不值得做。"
- AI："我理解你的意思是，你担心市场规模不足，投入产出可能不成比例，所以犹豫是否继续。我的理解正确吗？"
